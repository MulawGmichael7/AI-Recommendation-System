# Meme Trading PLC — AI Recommender

A hybrid recommendation engine for a meme-trading e-commerce platform, where
users buy, sell, wishlist, and trade meme items (templates, meme cards,
rarity-tiered collectibles, etc).

## How it works

The recommender blends two signals:

| Signal | Method | Solves |
|---|---|---|
| **Collaborative filtering** | Truncated SVD (matrix factorization) over an implicit feedback matrix (views/wishlists/purchases/trades, weighted by event type + blended with explicit ratings) | "Users like you also liked…" |
| **Content-based filtering** | TF-IDF over item category + tags + rarity, cosine similarity | Cold-start for new items/users, "more like this" |

Per-user blend weight (`alpha`) scales with how much interaction history a
user has — brand-new users lean almost entirely on content similarity /
popularity, established users lean more on collaborative signal. Total
cold-start users (never seen before) fall back to global popularity.

```

├── data/
```

## Quickstart

```bash
pip install -r requirements.txt

# 1. Generate synthetic data (swap in your real data by matching this schema)
python data/generate_data.py

# 2. Train the model
python src/train.py

# 3. Evaluate offline
python src/evaluate.py

# 4. Serve recommendations via API
python src/app.py
curl http://localhost:5000/recommend/1
curl http://localhost:5000/similar/42
```

## Data schema (swap in real data by matching this)

**users.csv**: `user_id, archetype, favorite_categories, signup_day, trust_score`

**items.csv**: `item_id, title, category, tags, rarity, price_usd, creator_user_id, listed_day`

**interactions.csv**: `user_id, item_id, event_type (view/wishlist/purchase/trade), rating (1-5 or blank), day`

## Using it in code

```python
import pickle
model = pickle.load(open("models/recommender.pkl", "rb"))

# top-10 recommendations for user 1
model.recommend(user_id=1, k=10)

# filter to a category
model.recommend(user_id=1, k=10, category_filter="stonks-finance")

# "more like this" for a product page
model.similar_items(item_id=42, k=10)
```

## Current offline performance

On the synthetic dataset (500 users, 300 items, 12k interactions),
leave-one-out evaluation gives **Hit Rate@10 ≈ 0.17**, roughly **5x** better
than the random baseline (~0.03). Swap in real interaction data and re-run
`evaluate.py` to see production-relevant numbers.

## Next steps for production

- Replace synthetic data with real interaction logs (schema above)
- Retrain on a schedule (e.g. nightly) as `train.py` is idempotent
- Add implicit negative feedback (e.g. "skipped"/"dismissed" events)
- A/B test `alpha` weighting and SVD component count
- Consider swapping SVD for ALS or a neural two-tower model once data volume grows
- Add caching / precomputed top-N per user in front of the Flask API for latency
